# spec

## 硬命令

1. 未被“当前软规范”允许的内容，不得实现。
2. 属于后续阶段且当前任务不直接依赖的代码、接口、页面、结构，不得提前创建。
3. 接口契约、数据库语义、返回层级、状态模型等高影响决策未明确时，不得自行定死，必须先确认。
4. 占位、骨架、路由、空返回、空实现，不得算业务完成。
5. `减肥计划拍照识别软件开发设计文档.md` 为只读基线，禁止修改。
6. 新增通用组件前，如有必要，优先评估成熟且可落地的开源组件方案；确认不合适后再手写。
7. 本次执行若偏离“当前软规范”，必须先写 `project_develop_log.md`，再决定是否更新本文件。
