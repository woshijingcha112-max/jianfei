## 四、接口定义

### 统一返回格式
```json
{ "code": 0, "msg": "success", "data": {} }
```

### 接口调用约定
```text
除上传/下载接口外，所有接口统一使用 POST。
查询类接口不再使用 query string，查询条件统一放入 JSON body。
默认 Content-Type: application/json
图片上传接口使用 multipart/form-data
```

### 接口列表

```text
// 拍照识别
POST /diet/photo/upload        上传图片 → 调百度AI → 返回识别食物列表+热量Tag

// 饮食记录
POST /diet/record/save         保存一餐记录（含食物列表，用户可修正后提交）
POST /diet/record/list         查询某日记录  body: { "date": "2024-01-15" }
POST /diet/record/delete       删除记录      body: { "recordId": 1 }

// 食物搜索
POST /food/search              按关键词搜索食物  body: { "keyword": "鸡蛋" }
POST /food/detail              食物详情+可做菜品  body: { "id": 1 }

// 今日热量汇总
POST /diet/stat/today          今日已摄入热量汇总  body: { "date": "2024-01-15" }

// 目标 & 体重
POST /goal/get                 获取目标配置
POST /goal/save                设置目标（体重/每日热量上限）
POST /weight/record            记录今日体重

// 生理期（留口）
POST /period/record            记录生理期开始日期
POST /period/list              查询生理期历史记录
```

### 饮食记录表
```sql
CREATE TABLE `t_diet_record` (
```

### 饮食明细表
```sql
CREATE TABLE `t_diet_item` (
  `id`           BIGINT       NOT NULL AUTO_INCREMENT,

  `weight_g`     DECIMAL(7,1) COMMENT '预估重量g',
```

### 食物热量库（核心表）

> 数据来源优先级：① 《中国食物成分表》（中国疾控）② 薄荷食物库 boohee.com/food ③ USDA FoodData Central。
> 数据由开发者导入，通过 `data_source` 字段标记来源，初期 300 条即可冷启动。

```sql
CREATE TABLE `t_food_library` (
  `id`            BIGINT       NOT NULL AUTO_INCREMENT,
  `food_name`     VARCHAR(100) NOT NULL COMMENT '食物名（中文）',
  `food_name_en`  VARCHAR(100) COMMENT '英文名，用于AI识别结果匹配',
  `alias`         VARCHAR(200) COMMENT '别名，逗号分隔，如"番茄,西红柿"',
  `category`      VARCHAR(50)  COMMENT '分类：蔬菜/水果/主食/肉类/海鲜/蛋奶/豆制品/坚果/零食/快餐/调味品',
 
  -- 每100g可食部营养成分
  `calories_kcal` DECIMAL(7,1) NOT NULL COMMENT '热量kcal/100g',
  `protein_g`     DECIMAL(6,2) COMMENT '蛋白质g',
  `fat_g`         DECIMAL(6,2) COMMENT '脂肪g',
  `carbs_g`       DECIMAL(6,2) COMMENT '碳水g',
  `fiber_g`       DECIMAL(6,2) COMMENT '膳食纤维g',
 
  -- Tag
  `tag_color`     TINYINT      NOT NULL COMMENT '1绿≤100 2橙101-300 3红≥301',
  `tag_reason`    VARCHAR(100) COMMENT '简短提示，如"高油高糖，建议少吃"',
 
  -- 辅助
  `edible_rate`   DECIMAL(5,2) DEFAULT 100.00 COMMENT '可食部比例%',
  `common_unit`   VARCHAR(50)  COMMENT '常见单位，如"1个约60g"',
  `data_source`   VARCHAR(100) COMMENT '数据来源备注',
 
  `created_at`    DATETIME     DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_name` (`food_name`),
  INDEX `idx_category` (`category`),
  FULLTEXT INDEX `ft_search` (`food_name`, `alias`)
);
```

## 六、Tag 颜色分级标准

依据：百度健康医学科普 + 中国居民膳食指南（GB 28050-2025）

| Tag | 热量（每100g）| 含义 | 典型食物 |
|-----|-------------|------|---------|
| 🟢 绿 | ≤ 100 kcal | 可多吃 | 黄瓜15、西红柿20、生菜13、冬瓜11、草莓32、豆腐81 |
| 🟠 橙 | 101~300 kcal | 适量 | 米饭116、鸡胸肉133、鸡蛋144、牛肉125、香蕉91 |
| 🔴 红 | ≥ 301 kcal | 少吃 | 炸鸡320、薯条320、蛋糕350+、五花肉395、坚果580+ |

> 坚果虽红，`tag_reason` 可备注"富含优质脂肪，少量食用无妨"

## 七、AI 识别接入（百度菜品识别）

```
接口：https://aip.baidubce.com/rest/2.0/image-classify/v2/dish
免费额度：每天500次
返回：食物名称 + 置信度
```

**后端处理逻辑：**
```
上传图片
  → 保存到 /data/diet-app/uploads/年/月/日/uuid.jpg
  → 调百度AI，取 probability > 0.4 的结果
  → 逐一查 t_food_library（精确匹配 → alias匹配 → 模糊匹配）
  → 匹配到：返回 food_id + 热量 + tag_color
  → 未匹配：返回食物名 + matched=false（前端提示手动搜索确认）
  → 整体结果返回给安卓端
```

## 十一、注意事项

1. 安卓上传前压缩到 **1MB 以内**（Kotlin 首期建议使用 Compressor，或自行封装压缩逻辑）
2. AI识别准确率约 70~80%，必须保留手动修正入口
3. 食物库冷启动先录 300 条家常菜/主食/蔬菜，外卖品牌后续补
4. 腾讯云安全组只开放 80/443/22 端口，其余全关
5. `/data/diet-app/uploads/` 定期备份（腾讯云快照或 crontab + rsync）

---

## 十二、开发约束补充

1. 本项目按 **单用户 MVP** 实现，仅服务固定使用者，后端默认按单用户处理，不设计注册、登录、鉴权、用户切换。
2. 用户相关数据可直接按固定用户处理，首期实现可约定 `user_id = 1`。
3. 本期优先跑通主流程，暂不扩展复杂业务实体关系，不引入额外业务域拆分。所有查询接口统一按 POST 处理。
4. AI 识别结果必须保留手动修正入口，识别仅作为辅助，不作为最终入库结果。
5. 腾讯云安全组、WAF、Nginx 安全加固等内容属于部署与运维范围，本期不纳入开发排期。
